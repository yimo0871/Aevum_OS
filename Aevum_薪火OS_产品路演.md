# Aevum / 薪火

> **Experience never fades. It compounds.**
> **经验不熄，代代相传。**

---

## 开场：一个不该被浪费的时代

> **你让 Agent 跑了一千次任务。**
> **你手里留下了什么？**
>
> 聊天记录。几个跑通的 prompt。零散的成功案例。
> **没有结构。没有积累。没有传承。**
>
> 每一次执行，都是一次性消耗品。用完即弃。
>
> 这不正常。

---

我们现在所处的时刻，是 AI Agent 的寒武纪大爆发——每天有数百万个 Agent 在执行任务、调用工具、生成结果。但所有这些执行都在做同一件事：**浪费经验**。

你让 Agent 花 3 个小时调试出一个完美的部署脚本，然后呢？对话结束，烟消云散。下一个人——甚至明天的你自己——面对同样的任务，从零开始。

**人类发明了文字来传承知识，发明了 Git 来协作代码，发明了 Wikipedia 来共享事实。但我们至今没有发明一种方式，让 AI Agent 的执行经验被沉淀、被复用、被全球共享。**

直到现在。

---

## 我们正在构建什么

**Aevum（薪火）**——人类历史上第一个以"经验"为第一公民的操作系统。

它不是又一个 Agent 框架。不是又一个 prompt 工具。不是又一个 workflow 引擎。

它是 Agent 时代的 **GitHub + Stack Overflow + Wikipedia 的融合体**，只不过它存储的不是代码、不是问答、不是百科词条，而是**经验图谱（Experience Graph）**——Agent 每一次执行的完整记录、评估和可复用模式。

---

## 为什么这是不可避免的

| 如果这个存在 | 会发生什么 |
|--------------|------------|
| Agent 执行任务前，先检索全球是否有相似经验 | 不再重复造轮子，跳过 90% 的试错 |
| 每一次成功执行，自动沉淀为可复用资产 | Agent 越用越聪明，形成复利效应 |
| 失败的经验同样被记录 | 他人的坑成为你的免费护栏 |
| 经验在全球 Agent 之间自由分叉、改进、合并 | 形成经验的"开源社区"，集体智能持续进化 |
| 人类的角色从"操作者"变为"策展人" | 人类负责定义方向和判断结果，Agent 负责执行和沉淀 |

---

## 一句话

> **AI 不会取代人类。但没有经验记忆的 AI，会被有经验记忆的 AI 取代。**
>
> Aevum 就是那个记忆。
